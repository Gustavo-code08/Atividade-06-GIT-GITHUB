# Atividade-06-GIT-GITHUB
## Esse projeto tem como objetivo: Nos apresentar o GIT e o GITHUB, e nos mostrar como funcionam.

# O que é Git?

O Git é um sistema de controle de versão criado para acompanhar alterações em arquivos e projetos de software. Ele permite salvar versões do código, recuperar mudanças antigas e colaborar com outras pessoas sem sobrescrever o trabalho dos colegas.

# Principais funções do Git
## Controle de versões
Guarda o histórico completo do projeto, permitindo voltar para versões anteriores quando necessário.
Commits
Salva “snapshots” do projeto em determinados momentos. Cada commit registra o que mudou, quem mudou e quando mudou.
## Branches (ramificações)
Permitem criar versões paralelas do projeto para desenvolver funcionalidades sem afetar a versão principal.
Merge
Junta alterações feitas em diferentes branches.
## Clone
Faz uma cópia completa de um repositório remoto para o computador local.
## Push e Pull
git push: envia alterações locais para o repositório online.

git pull: baixa atualizações do repositório remoto.
## Trabalho colaborativo
Várias pessoas podem trabalhar simultaneamente no mesmo projeto.
# O que é GitHub?

O GitHub
 é uma plataforma online que hospeda repositórios Git na nuvem. Ele funciona como uma rede social para desenvolvedores e também como plataforma de colaboração de projetos.

# Principais funções do GitHub
## Hospedar projetos online
Permite armazenar repositórios na internet.
## Colaboração em equipe
Desenvolvedores podem contribuir juntos no mesmo projeto.
## Pull Requests
Sistema para revisar e aprovar alterações antes de integrá-las ao projeto principal.
## Issues
Ferramenta para registrar bugs, tarefas e melhorias.
## Portfólio profissional
Muitos recrutadores avaliam projetos publicados no GitHub.
## Documentação do projeto
Uso de arquivos README.md para explicar como instalar e usar o projeto.
# Passo a passo para postar um projeto no GitHub
## Introdução
Este tutorial mostrará como carregar um grupo de arquivos em um repositório do GitHub.

Ao carregar arquivos em um repositório do GitHub, o seguinte é possível:

Aplique o controle de versão ao fazer edições nos arquivos, para que o histórico do projeto seja protegido e gerenciável.
Faça backup do seu trabalho, porque seus arquivos agora estão armazenados na nuvem.
Fixe o repositório no seu perfil pessoal, para que outras pessoas possam ver o seu trabalho.
Compartilhe e discuta seu trabalho com outras pessoas, seja pública ou privadamente.
Se você já conhece o Git e deseja carregar um repositório Git armazenado localmente no GitHub, consulte https://docs.github.com/pt/migrations/importing-source-code/using-the-command-line-to-import-source-code/adding-locally-hosted-code-to-github#adding-a-local-repository-to-github-using-git

## Pré-requisitos
Você precisa ter uma conta do GitHub. Para saber mais, confira https://docs.github.com/pt/get-started/start-your-journey/creating-an-account-on-github
Você deve ter um grupo de arquivos que gostaria de carregar.
## Etapa 1: Criar um novo repositório para seu projeto
É uma boa ideia criar um novo repositório para cada projeto individual em que você está trabalhando. Se você estiver escrevendo um projeto de software, agrupar todos os arquivos relacionados em um novo repositório facilita a manutenção e o gerenciamento da base de código ao longo do tempo.

No canto superior direito de qualquer página, selecione + e clique em Novo repositório.

Na caixa "Nome do repositório", digite um nome para o projeto. Por exemplo, digite "meu-primeiro-projeto".

Na caixa "Descrição", digite uma breve descrição. Por exemplo, digite "Este é meu primeiro projeto no GitHub".

Selecione se o repositório será Público ou Privado. Selecione "Público" se quiser que outras pessoas possam ver seu projeto.

Selecione Adicionar um arquivo LEIAME. Você editará esse arquivo em uma etapa posterior.

Clique em Criar repositório.

## Etapa 2: Fazer upload de arquivos para o repositório do seu projeto
Até agora, você só deve ver um arquivo listado no repositório, o arquivo README.md que você criou quando inicializou o repositório. Agora, vamos carregar alguns dos seus próprios arquivos.

À direita da página, selecione o menu suspenso Adicionar arquivo.
Clique em Carregar arquivos no menu de opções.
No computador, abra a pasta que contém o trabalho e arraste e solte todos os arquivos e pastas no navegador.
Na parte inferior da página, em "Confirmar alterações", selecione "Confirmar diretamente na ramificação main e clique em Confirmar alterações.

## Etapa 3: Editar o arquivo LEIAME para o repositório do seu projeto
O arquivo LEIAME do repositório é normalmente o primeiro item que alguém verá ao visitá-lo. Ele geralmente contém informações sobre o que é o seu projeto e por que o seu projeto é útil.

Como aprendemos no tutorial Olá, Mundo, o arquivo LEIAME (README.md) é escrito na sintaxe Markdown. O Markdown é uma linguagem fácil de ler e de gravar para formatar texto sem formatação.

Nesta etapa, editaremos o README.md do seu projeto usando Markdown para que ele inclua algumas informações básicas sobre o projeto.

Na lista de arquivos, clique em README.md para exibir o arquivo.

No canto superior direito da exibição de arquivo, clique no "lápis" para abrir o editor de arquivos.

Você verá que algumas informações sobre seu projeto foram pré-preenchidas para você. Por exemplo, você deve ver o nome do repositório e a descrição do repositório que você concluiu na Etapa 1 exibidos na linha 1 e na linha 2.
Exclua o texto existente além do # e digite um título adequado para o projeto.

Exemplo: # About my first project on GitHub.
Em seguida, adicione algumas informações sobre seu projeto, como uma descrição da finalidade do projeto ou seus principais recursos.

*Observação*

Se você não tiver certeza do que escrever, confira outros repositórios no GitHub para entender como outras pessoas descrevem os próprios projetos.

Para aplicar formatação mais sofisticada, como adicionar imagens, links e notas de rodapé, confira https://docs.github.com/pt/get-started/writing-on-github/getting-started-with-writing-and-formatting-on-github/basic-writing-and-formatting-syntax

Acima do novo conteúdo, clique em Visualizar.

Veja como o arquivo será renderizado depois que salvarmos nossas alterações e, em seguida, alterne novamente para "Editar".

Continue a editar e visualizar o texto até o conteúdo do LEIAME estar satisfatório.

No canto superior direito, clique em Confirmar alterações.

Na caixa de diálogo que é aberta, uma mensagem do commit foi pré-preenchida para você ("Atualizar LEIAME.md") e, por padrão, a opção "Confirmar diretamente na ramificação main" foi selecionada. Deixe essas opções como estão, siga em frente e clique em Confirmar alterações.

# Conclusão
Você criou um novo repositório, carregou alguns arquivos nele e adicionou um projeto README.

Se você definir a visibilidade do repositório como "Público", o repositório será exibido em seu perfil pessoal e você poderá compartilhar a URL do repositório com outras pessoas.

À medida que você adiciona, edita ou exclui arquivos diretamente no navegador em GitHub, GitHub rastreará essas alterações ("confirmações"), para que você possa começar a gerenciar o histórico e a evolução do seu projeto.

Ao fazer alterações, lembre-se de que você pode criar uma nova ramificação a partir da ramificação main do repositório, para que possa experimentar sem afetar a cópia principal dos arquivos. Em seguida, quando estiver satisfeito com um conjunto de alterações, abra uma solicitação de pull para mesclar as alterações em sua ramificação main. Para obter um lembrete de como fazer isso, confira Olá, Mundo.

# Resumo e comandos
# Git

Sistema de controle de versão usado para:

salvar histórico do código;

controlar alterações;

trabalhar com branches;

colaborar em projetos.

## Comandos principais:

git init              -- inicia repositório

git clone URL         -- clona projeto

git status            -- mostra estado dos arquivos

git add .             -- adiciona mudanças

git commit -m "msg"   -- salva alterações

git pull              -- baixa atualizações

git push              -- envia alterações

git branch            -- lista branches

git switch -c nome    -- cria branch

git merge nome        -- junta branches

git log               -- histórico

git diff              -- diferenças


# GitHub

Plataforma online para hospedar repositórios Git.

## Recursos principais:

hospedagem de código;

Pull Requests;

Issues;

colaboração em equipe;

GitHub Actions (automação);

documentação e releases.

## Fluxo básico:

git add .

git commit -m "alteração"

git push


## Diferença
Git = ferramenta de versionamento local.

GitHub = plataforma online para compartilhar e colaborar usando Git.
